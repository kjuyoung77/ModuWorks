# 백그라운드 감시기 무장 (단일 인스턴스 계약)

파일·폴더 변화를 감지하는 감시기를 **정확히 하나만** 돌린다. 중복 감시기는 중복 알림·경쟁을 부른다.

## 단일 인스턴스 계약 (P1)
1. **원자 락으로 직렬화** — `mkdir`(존재 시 실패 = 배타 뮤텍스)로 획득 임계구역을 만든다. `mv -f`는 배타 lock이 아니라 동시 시작에서 둘 다 통과할 수 있으므로 쓰지 않는다.
2. **락 안에서 lease 재검사** — 획득 후 lease를 다시 본다. 살아있으면(재사용) 락을 풀고 종료, stale/부재면 내 lease를 쓴다.
3. **owner token으로 소유권 표기** — lease에 고유 `owner`(PID + 시작시각 + 난수) + PID + 시작시각 + scope를 기록한다.
4. **PID 실존 + mtime 신선도로 생존판정** — `kill -0` 로 PID 실존 + heartbeat mtime이 폴링 주기 × 2 이내여야 '살아있음'. **살아있는 lease는 절대 선점하지 않고 stale만 회수.**
5. **owner 일치 시에만 cleanup** — 종료(`trap ... EXIT INT TERM`) 시 lease의 owner가 나와 같을 때만 지운다(남의 lease 삭제 방지). 루프 중 owner가 바뀌면 스스로 종료(좀비 이중감시 방지).

## 변경 감지 (재귀)
top mtime만 보면 폴더 내부 파일 수정을 놓친다. **재귀 스냅샷**(하위 전체의 타입·경로·크기·mtime 해시)으로 비교 → 기존 파일 수정·신규·rename·삭제·하위경로 변경을 모두 감지한다.

## 결정론적 헬퍼
설명을 매번 반복하지 말고 동봉한 `scripts/ensure_single_watcher.sh`를 쓴다.
⚠️ 공개 저장소이므로 헬퍼에 **비공개 경로·키·토큰을 넣지 않는다** — 감시 경로·스코프는 **인자로 주입**한다.

```
scripts/ensure_single_watcher.sh <감시대상경로> <lease파일경로> [폴링초=15]
```

> 왜: "감시기 켜"를 매번 손으로 짜면 lease·PID·mtime 처리를 빠뜨려 좀비·중복이 생긴다. 계약을 스킬 + 헬퍼로 두면 한 줄 요청으로 동일한 안전 형태를 재현한다.
