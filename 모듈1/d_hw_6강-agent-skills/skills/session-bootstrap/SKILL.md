# 세션 부트스트랩 (연속성 오케스트레이터)

새 세션/재개/재부팅 후 **첫 작업**. 아래 순서를 지킨다 — 시작하자마자 무조건 "재무장"하면 감시기가 중복 생성된다.

1. **현재 상태 읽기** — 상태판·핸드오프·메모리를 먼저 읽어 맥락을 잡는다. (이 단계에서는 아무것도 켜지 않는다.)
2. **기존 감시기 생존·소유권 확인** — heartbeat/lease 파일을 보고 살아있는 감시기가 있는지 확인한다. 생존 판정은 표시 시각이 아니라 **파일 mtime/epoch**로(폴링 주기 × 2 초과 = stale).
3. **필요할 때만 무장** — 살아있는 감시기가 있으면 **재사용**한다. 없거나 stale일 때만 `watcher-arm`을 호출한다.
4. **미처리 스레드 복원 (복합키)** — 열린 스레드를 **(스레드ID + 차수 + 발신→수신 방향)** 복합키로 판정한다. `회신요망=예` 텍스트만으로 열림 단정 금지 — 반대 방향의 다음 차수 회신(`in_reply_to`/원참조 ID로 대응 확인)이 있으면 닫힘, `closed/superseded/withdrawn` 태그 반영, 같은 코드의 과거 차수와 최신 차수를 섞지 않는다(최신 우선·dedup).
5. **최신 결정 확인** — 가장 최근 결정 블록으로 방향을 맞춘다.
6. **브리핑** — 복원 결과 + 열린(미처리) 스레드 + "지금 할 일"을 3~5줄로 요약한다.

> 왜: 세션 경계마다 (a) 감시기 중복 생성, (b) 이미 처리된 스레드를 다시 열기가 흔하다. 이 순서를 스킬로 고정하면 중복 없이, 놓친 것 없이 이어받는다.

**호출:** `/session-bootstrap` (Claude Code 슬래시). 최초로 최상위 `.claude/skills/`를 만든 직후엔 도구 재시작이 필요할 수 있다.
