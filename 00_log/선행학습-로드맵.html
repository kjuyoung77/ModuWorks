2. 선행학습 우선순위

1순위 · Git / GitHub / 터미널

수업 중 가장 자주 발목을 잡는 것은 AI 이론보다 경로, 설치, Git, 제출 문제일 가능성이 큽니다.

  • cd, ls, mkdir, rm, cp, mv 같은 터미널 기본
  • Git 흐름: clone → add → commit → push → pull
  • GitHub 저장소 생성, README 작성, .gitignore 이해
  • 에러 로그를 복사해서 AI에게 정확히 질문하는 법

2순위 · Python 기본기

  • 변수, 리스트, 딕셔너리
  • 조건문, 반복문, 함수
  • 파일 읽고 쓰기
  • pip install, 가상환경 개념
  • 에러 메시지 읽기

목표는 “알고리즘 고수”가 아니라, 수업 코드와 AI가 만들어준 코드를 읽고 고칠 수 있는 수준입니다.

3순위 · 웹 기초

  • HTML: 화면 구조
  • CSS: 배치와 스타일
  • JavaScript: 클릭, 입력값 처리
  • API: 요청과 응답
  • JSON: 데이터 교환 형식
  • 서버 기본: 라우팅, 요청 처리

4순위 · LLM / 프롬프트 / RAG 감 잡기

  • LLM, 프롬프트, 컨텍스트, 토큰
  • RAG: 내 문서를 검색해 답변에 붙이는 방식
  • 임베딩: 문장을 숫자 벡터로 바꾸는 것
  • 벡터DB: 의미 기반 검색 저장소
  • 에이전트: LLM이 도구를 호출하며 작업하는 구조
  • MCP: AI가 외부 도구·데이터와 연결되는 표준화된 방식
추천 순서: Git·터미널 → Python → 웹/API → LLM·RAG 용어 → 그다음 NLP/CV/논문 순서가 가장 안전합니다.

3. 4주 선행학습 루틴

기간할 것목표
1주차Git / GitHub / 터미널 + Python 기본과제 제출, 저장소 관리, 환경 문제 대응
2주차HTML / CSS / JavaScript / API / JSONAI가 만든 웹앱 코드를 읽고 수정
3주차Python으로 작은 자동화 만들기파일 정리, 간단 API 호출, 데이터 저장
4주차LLM API / 프롬프트 / RAG 개념에이전트·RAG 수업 진입 준비
하루 1~2시간만 가능하다면 “개념 영상 보기 30분 + 직접 쳐보기 60분 + 오늘 배운 것 기록 10분” 정도가 좋습니다.

4. 추천 미니 프로젝트 3개

출결·과제 정리 도우미

  • 오늘 할 일 입력
  • 마감일 입력
  • 완료 체크
  • 로컬 JSON 파일에 저장

Git, Python, JSON, 간단한 UI를 함께 연습할 수 있습니다.

강의자료 요약 도우미

  • 텍스트 붙여넣기
  • 핵심 요약
  • 모르는 용어 추출
  • 질문 리스트 생성

프롬프트 설계와 LLM 활용 감각을 키우기 좋습니다.

나만의 학습 로그 앱

  • 오늘 배운 것
  • 막힌 것
  • 해결한 것
  • 내일 할 것
  • GitHub에 매일 커밋

포트폴리오와 면접 스토리의 씨앗이 됩니다.

작게 만들고, 매일 고치고, GitHub에 남기는 것이 가장 좋은 선행학습입니다.

5. 매일 학습법

수업 전 20분
오늘 나올 키워드 3개만 미리 검색합니다. 깊게 파지 말고 단어의 뜻과 쓰임만 확인합니다.
수업 중
모르는 내용을 전부 정리하려 하지 말고, “지금 막힌 지점”과 “나중에 다시 볼 키워드”를 분리합니다.
수업 후 40~60분
오늘 배운 내용을 아주 작은 코드나 문서로 재현합니다. 재현이 안 되면 그 부분만 다시 질문합니다.
하루 마감
학습 로그를 남기고 GitHub에 커밋합니다. 커밋 메시지는 “무엇을 이해했는지”가 보이게 씁니다.

6. 지금 깊게 안 해도 되는 것

커리큘럼에 CV, NLP, HuggingFace, BERT, Machine Translation, Chatbot, 논문 작성이 나오지만 첫날부터 깊게 들어가면 오히려 부담이 커질 수 있습니다.

  • 수식 중심 머신러닝·딥러닝 이론
  • Transformer 논문 완독
  • NLP/CV 모델 구조 세부 구현
  • 배포·클라우드·CI/CD 고급 설정
  • 논문 작성법 전체
지금 목표: “잘 모르지만 프로젝트를 따라 만들 수 있는 상태”를 먼저 만드는 것입니다. 깊은 이론은 수업 흐름에 맞춰 필요한 순간에 들어가도 늦지 않습니다.

7. 선행학습 체크리스트

  • 터미널에서 폴더 이동과 파일 생성·삭제를 할 수 있다.
  • GitHub 저장소를 만들고 commit/push 할 수 있다.
  • Python 리스트·딕셔너리·함수를 읽고 쓸 수 있다.
  • 에러 메시지를 복사해 원인과 해결방법을 질문할 수 있다.
  • HTML/CSS/JS가 각각 무엇을 담당하는지 설명할 수 있다.
  • API 요청과 JSON 응답의 구조를 대략 이해한다.
  • LLM, 프롬프트, 토큰, 컨텍스트, RAG, 임베딩, 벡터DB, 에이전트, MCP의 의미를 한 문장으로 설명할 수 있다.
  • 작은 미니 프로젝트 하나를 GitHub에 남기기 시작했다.
첫달의 목표는 “완벽한 이해”가 아니라 “수업을 놓치지 않는 작동 가능한 기초 체력”입니다.
이 문서는 수업 로드맵과 첫날 안내 내용을 바탕으로 만든 개인 선행학습 가이드입니다.
