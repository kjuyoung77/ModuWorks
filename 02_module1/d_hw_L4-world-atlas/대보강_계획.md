## 0. 현재 상태 정밀 진단

### 사이트 뎁스(정보 구조)
현재는 **딱 2뎁스**뿐이다.

- **뎁스 1 — `index.html`(목록):** 40개국 카드 그리드, 검색, 지역 필터(7), 즐겨찾기, 인기순, TOP5 조회 밴드, 회전 도트 지구본.
- **뎁스 2 — `country.html`(상세):** 아래 11개 섹션.

### 상세페이지 현재 11개 섹션
1. 히어로(국기·이름·지역·위키 사진)
2. 브레드크럼(Atlas / 지역 / 나라)
3. 개요(한 줄 리드 + 위키 요약문단)
4. 팩트 그리드(수도·인구·면적·언어·종교·통화·국제전화·조회수)
5. 상대 규모 시각화(인구·면적 로그 막대)
6. 국기 이야기(`FLAG_MEANING`)
7. 나의 기록(가봤어요·가고싶어요·즐겨찾기)
8. 대표 명소(**1곳**, 위키)
9. 알아두면(재미 사실 1개)
10. 댓글(Supabase)
11. 같은 지역(관련국)

### 데이터 소스
- **정적(`data.js`):** 40개국 × 약 16필드(이름/영문/수도/인구/면적/언어/종교/통화/국제전화/설명/랜드마크/heroPos 등), `FLAG_MEANING`, `SEED_COMMENTS`, i18n(ko/en).
- **동적(런타임):** Wikipedia REST(국가 요약문단 + 랜드마크 요약·이미지).
- **백엔드(Supabase):** 방문통계·나라별 조회수·반응·즐겨찾기·댓글(RLS).

### 한계(=보강 포인트)
- 상세페이지가 **얕다**: 역사·지리·문화·경제·자연·사회가 없다. "재미 사실 1개 + 명소 1곳"이 깊이의 전부.
- **뎁스가 2개뿐**: 나라를 '하나씩만' 본다. 비교·랭킹·지역·테마·지도 탐색 같은 **다른 관점의 뎁스가 없다.**
- 데이터가 손으로 넣은 16필드에 묶여 확장이 어렵다 → **무료 API로 자동 보강** 필요.

## 1. 데이터 전략 (이 계획의 심장)

40개국에 역사·지리·문화·경제를 손으로 40번 쓰는 건 비현실적·유지불가. **무료·키불요·CORS 허용 소스**로 자동 보강한다.

| 소스 | 주는 것 | 비고 |
|---|---|---|
| **REST Countries** (restcountries.com) | 인접국(borders)·소지역(subregion)·시간대·통화·언어·국제전화·차량통행방향·수도좌표·지역·국장(coat of arms)·지도링크·gini | 무키·CORS·구조화 최고 |
| **Wikidata** (REST/SPARQL) | HDI·GDP·정부형태·국가원수·독립일·ISO코드·UNESCO 등재수·최고봉 등 | 구조화 심화 |
| **Wikipedia REST** | 프로즈(문단)·이미지 — `History of X`·`Culture of X`·`Geography of X`·`Economy of X`·`Cuisine of X` 요약 | 이미 쓰는 패턴 확장 |
| (선택) **World Bank API** | GDP·기대수명·도시화율 등 지표 시계열 | 무키 |
| (선택) **Open-Meteo** | 수도 기후/현재날씨 | 무키 |

### 권장 아키텍처 — 3계층 + 빌드타임 프리페치
- **A계층(즉시·정본):** `data.js` 히어로 팩트(현행 유지, 소수 필드 추가).
- **B계층(구조화):** REST Countries + Wikidata → **빌드타임 1회 수집해 `atlas-data.json`으로 저장소에 커밋**. → 런타임 API 의존 0, 오프라인·채점 안전, 초고속.
- **C계층(프로즈·이미지):** Wikipedia REST를 **런타임 fetch**(주제별 요약) + `localStorage` 캐시. 최신성 확보, 실패 시 A계층으로 폴백(현행 패턴).

> ★ 핵심 권고: **구조화 사실은 커밋된 JSON(안정)**, **읽을거리·사진은 런타임(신선)**. 채점·데모 신뢰성과 콘텐츠 풍부함을 둘 다 잡는다.

## 2. 상세페이지 전문화 — 새 섹션 청사진

긴 페이지 + **상단 고정 서브내비(탭)** 로 스캔 가능하게(모바일은 접힘). 다크 에디토리얼 톤·Fraunces 세리프 유지.

| # | 섹션 | 담을 것 | 주 소스 |
|---|---|---|---|
| 1 | **개요 Overview** | 현행 리드 + 위키 인트로(유지) | data + Wiki |
| 2 | **한눈에 At a glance** | 팩트 확장: 소지역·**인접국(링크)**·시간대·차량통행·ISO·국장 | REST Countries |
| 3 | **지리·자연 Geography** | 위치·지형·기후·해안선·최고봉·생물군계 + **미니 지도** + 인접국 | Wiki `Geography of` + Wikidata |
| 4 | **역사 History** | 핵심 연표(고대→근대→현대)·독립일·주요 사건 | Wiki `History of` + Wikidata |
| 5 | **문화 Culture** | 언어·종교 상세·요리·예술·축제·에티켓 | Wiki `Culture of`·`Cuisine of` |
| 6 | **사람·사회 People** | 인구 상세·HDI·기대수명·도시화·인구 구성 | Wikidata·World Bank |
| 7 | **경제 Economy** | GDP·주력산업·통화·수출품 | Wikidata·World Bank |
| 8 | **명소 Places** | **여러 곳**(현재 1곳→3~6곳) 갤러리 | Wiki 관련/지오서치 |
| 9 | **여행 노트 Travel** | 베스트 시즌·기본 회화·일반 안전 팁 | 큐레이션 + API |
| 10 | **국기 이야기** | 현행(유지) | data |
| 11 | **알아두면 / 나의 기록 / 댓글 / 관련국** | 현행(유지·명소 아래로 재배치) | data + Supabase |

- 섹션마다 **출처 표기**(신뢰), 데이터 없으면 그 섹션 **우아하게 숨김**(현행 폴백 원칙).
- 상단 서브내비: `개요·지리·역사·문화·사람·경제·명소·여행` 앵커 스크롤.

## 3. 새 섹션/뎁스 후보 — 정밀 분석

"나라 하나씩 보기" 외에 **다른 관점의 뎁스**를 신설. 각 항목 = 가치/노력/데이터/백엔드 표기.

### A. 탐색 뎁스(목록↔상세 사이)
1. **지역 허브 `region.html`** — 대륙별(아시아·유럽…) 소개·지도·소속국 그리드·지역 통계·최고기록. *(가치 高 / 노력 中 / 기존 REGION 데이터)*
2. **인터랙티브 지도/지구본 내비** — 도트 지구본(이미 있음)이나 SVG 세계지도를 **클릭→나라 이동**으로. 시그니처 탐색. *(가치 高 / 노력 中~高)*

### B. 분석·비교 뎁스
3. **랭킹·차트 `rankings.html`** — 인구·면적·인구밀도·GDP·HDI 등 **정렬 리더보드 + 데이터 시각화**. *(가치 高 / 노력 中 / B계층 데이터)*
4. **비교 `compare.html?a=fr&b=jp`** — 두(또는 N개) 나라를 팩트·막대로 **나란히**. 몰입도 큰데 데이터 비용 낮음. *(가치 高 / 노력 中)*

### C. 에디토리얼·발견 뎁스
5. **테마 컬렉션 `collections.html`** — 큐레이션 묶음: 섬나라·다국어 국가·UNESCO 부자·커피 원산지·비식민 국가·초소형국가 등. 다크 에디토리얼 톤과 최적. *(가치 中~高 / 노력 中)*
6. **오늘의 나라 / 랜덤** — 매일 대표국·랜덤 버튼. 재방문 유도. *(가치 中 / 노력 低)*

### D. 참여·게이미피케이션 뎁스
7. **퀴즈/게임 `quiz.html`** — 국기 맞히기·수도 맞히기·"이 나라는?" + **Supabase 점수판**. *(가치 高 / 노력 中~高 / 백엔드 신규 테이블)*
8. **나의 아틀라스/여권 `passport.html`** — 가봤어요·가고싶어요·즐겨찾기를 **개인 대시보드**로 집계 + 방문 지도 + 통계. 기존 백엔드 재사용(신규 테이블 불필요). 리텐션 강함. *(가치 高 / 노력 中)*

### E. 기반
9. **전역 상단 내비** — 현재 상세페이지에만 뒤로가기뿐. `Atlas·Map·Regions·Rankings·Compare·Collections·Quiz·My Atlas` 전역 내비 신설.
10. **검색 고도화** — 나라뿐 아니라 수도·명소·팩트까지 통합 검색.

- **재사용:** 여권/나의 아틀라스 = 기존 `country_marks`·`favorites` 집계만(신규 불필요).
- **신규(선택):** 퀴즈 점수판(`quiz_scores`), 컬렉션 저장(`collection_saves`). 모두 RLS.

## 5. 단계별 로드맵 (권장 우선순위)

| Phase | 내용 | 산출물 | 리스크 |
|---|---|---|---|
| **P1 · 기반 + 최대효과** | ① 데이터 보강 파이프라인(REST Countries+Wikidata→커밋 JSON) ② 상세페이지 전문화(지리·역사·문화·경제·사람·명소多 + 서브내비) ③ 전역 상단 내비 | 상세페이지 백과화 | API 신뢰성→커밋 JSON+폴백으로 해소 |
| **P2 · 새 관점 뎁스** | 랭킹·차트 / 비교 / 지역 허브 | 3개 신규 페이지 | 데이터 시각화 품질 |
| **P3 · 참여·발견** | 인터랙티브 지도 내비 / 테마 컬렉션 / 퀴즈+점수판 / 나의 아틀라스(여권) | 탐색·게이미피케이션 | 백엔드 신규 테이블·범위 |

- 각 Phase는 **독립 배포 가능**. Phase마다 로컬 검증→승인→배포.
- 노드 과제(제출물) 관점: P1만으로도 "전문 백과 + 실데이터 파이프라인 + 백엔드"로 이미 강력.

## 6. 디자인 원칙(현행 다크 에디토리얼 계승)
- Fraunces 세리프 악센트 + Space Grotesk, 헤어라인 구획, 깊은 다크 팔레트.
- 상세 서브내비 = 스티키 탭. 차트 = 미니 SVG(막대/도넛/미니맵). 모바일 반응형·접근성 유지.
- 콘텐츠·규정·데이터 손실 0(문서편집 원칙), 출처 표기.

## 7. 결정 필요 포인트
1. **데이터 방식:** (A) 빌드타임 수집→커밋 JSON〔권장·안정〕 vs (B) 순수 런타임 fetch〔신선·리스크〕.
2. **전문화 깊이:** 주제별 큐레이션 2~3문장 vs 위키 섹션 통 요약.
3. **범위/순서:** P1만 먼저 vs P1~P3 전부 그린라이트.
4. **상세 레이아웃:** 스티키 탭 서브내비(권장) vs 단순 롱스크롤.

## 부록 — 즉시 착수 가능한 P1 세부
- `enrich.mjs`(1회 실행): REST Countries+Wikidata fetch→`atlas-data.json` 커밋.
- `country.html`: 서브내비 + 신규 섹션 렌더 함수(있으면 표시/없으면 숨김) + 주제별 Wikipedia 런타임 fetch.
- `data.js`: 인접국 코드 매핑 헬퍼(borders→상세 링크).
- 캐시버스터 `?v=` 상향, 두 문서 수치 정합성 유지.
