# 노드 4 프로젝트 제출 런북 — 세계 국가 도감 v2

갈래 B. 프론트(26개국 도감) + **Supabase 백엔드: 방문자 수 · 나라별 조회수(인기순위) · 댓글 · 가봤어요/가고싶어요 · 즐겨찾기**(사용자별, RLS 보안). 화면은 **한국어/영어 토글**(KO/EN) 지원.
코드는 다 되어 있고 **★ 표시 단계만 직접** 하면 됩니다.

## 설계 한 장 (제출물에 포함)
| 항목 | 내용 |
|---|---|
| 백엔드가 꼭 필요한 기능 | **저장·집계** — 방문자 수·조회수는 서버가 모든 사용자 걸 모아 세야 하고, 댓글·반응·즐겨찾기는 새로고침·다른 기기에도 남아야 함 |
| 화면만으로 되는 기능 | 목록·검색·지역/즐겨찾기 필터·상세·**언어 토글(i18n)** (프론트에서 동작) |
| 창구(엔드포인트) | `comments`·`country_marks`·`favorites` = CRUD / `page_views`(방문·조회 로그 insert) + 집계 함수 **RPC** `get_visit_stats()`·`get_view_counts()` |
| 사용자 구분 | Supabase **익명 로그인** → `auth.uid()` 로 본인 것만 수정(RLS) |
| 프라이버시 | `page_views` 원본 로그는 **SELECT 정책 없음**(직접 조회 불가) → 집계는 `SECURITY DEFINER` 함수로 숫자만 공개 |
| 다뤄야 할 비밀값 | publishable(공개) 키만 프론트에 / service_role(비밀) 키는 절대 금지 |
| 호스팅 | 백엔드=Supabase, 프론트=GitHub Pages(무료) |

## ★1. (이미 함) Supabase 프로젝트 — world-atlas 생성 완료

## ★2. 새 SQL 실행 (표 4개 + 집계 함수 2개)
SQL Editor → New query → [`SUPABASE_설정.sql`](SUPABASE_설정.sql) 전체 붙여넣고 **Run** → "Success".
(댓글·반응·즐겨찾기·**방문/조회 로그** 표 + RLS 정책 + **집계 RPC 함수 2개**가 만들어짐. 이전 표는 그냥 둬도 무방 — `create ... if not exists`라 재실행 안전)

## ★3. 익명 로그인 켜기  ← v2에서 새로 필요!
좌측 **Authentication → Sign In / Providers**(또는 Settings) → **Anonymous sign-ins** 를 **켜기(Enable)** → 저장.
> 이게 꺼져 있으면 댓글·즐겨찾기가 "로그인 실패"로 안 됩니다. 앱 상단 안내줄에 실패 메시지가 뜨면 여기부터 확인.

## ★4. 키 넣기
API Keys 페이지에서 **Publishable key**(`sb_publishable_...`, 브라우저 안전) 복사 →
[`data.js`](data.js) 위쪽 이 줄의 따옴표 안만 교체:
```js
const SB_KEY = "여기에_PUBLISHABLE_KEY_붙여넣기";
```
→ 저장. (Project URL은 이미 넣어둠. ⚠️ Secret key는 넣지 말 것. index.html·country.html이 이 data.js 하나를 공유)

→ 여기까지 하면 **로컬에서 백엔드 동작**: index.html 열면 상단에 "👁 누적 방문 N · 오늘 N"이 뜨고(연결 성공), 카드에 👁 조회수 배지·🔥 인기순·상단 "가장 많이 본 나라"가 채워짐. 상세에서 가봤어요·댓글·별(즐겨찾기) 눌러보고, 새로고침해도 남으면 OK.

## ★5. GitHub Pages로 배포 (제출용 URL)
게임 때처럼:
1. 이 폴더(`노드3-프레임워크-세계국가도감`)를 **새 공개 저장소**로 GitHub에 올림 (Claude Code에게 "이 폴더 새 공개 repo로 올려줘" 하면 로그인 빼고 도와줌)
2. 저장소 **Settings → Pages → Source: main / root** → Save
3. 1~2분 뒤 `https://내아이디.github.io/저장소이름/` 주소 생성

> Supabase는 기본적으로 모든 출처(origin)의 요청을 허용하므로, GitHub Pages 주소에서도 바로 백엔드가 동작합니다(별도 CORS 설정 불필요).

## ✅ 제출 전 보안 체크 (루브릭 ①)
- [x] 공개(publishable) 키만 프론트에, **service_role 비밀키 안 씀**
- [x] **RLS 켜짐** — 댓글·반응·즐겨찾기·방문로그 모두 정책 있음
- [x] **본인 것만 수정** — insert/delete는 `auth.uid() = user_id` 로 제한(인가)
- [x] **방문 로그 프라이버시** — `page_views`는 직접 SELECT 불가, 집계는 `SECURITY DEFINER` 함수(숫자만)로만 노출
- [x] 입력 제한 — 닉네임 20자·댓글 300자(DB check), 출력은 이스케이프(XSS 방지)
- [x] 무료 티어 한도 인지 — Supabase 무료(500MB, 7일 미사용 시 일시정지)
