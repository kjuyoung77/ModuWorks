# 비가역 실행 게이트 (위험등급 + 승인 결속)

"돌이킬 수 없는 것을 먼저 확인한다." 실행 직전, 위험을 **등급으로 나눠** 판단한다 — 일반 push와 force-push·prod 배포를 같은 위험으로 취급하지 않는다.

## 위험등급
| 등급 | 예 | 처리 |
|---|---|---|
| **낮음** | 로컬·재생성 가능 작업, 검증 통과 콘텐츠 배포 | 자율 진행 + 사후 기록 |
| **중간** | 일반 `git push`, 휴지통 이동(복구 가능), staging 배포 | 승인 |
| **높음(고위험 분기)** | `force-push` · **영구 삭제** · **production** 배포 · 결제/대금 · 저작권·권리 · 개인정보 · 비밀값 · DB migration | 명시 승인 + 별도 확인 |

**여러 등급이 동시에 해당하면 최고 등급을 적용한다** (예: production에 force-push = "높음"으로 처리).

## 승인 결속
승인은 다음에 **결속**한다: `대상 · 환경 · 동작 · 정확한 SHA/버전 · 영향 · 복구법 · 승인자 · 승인 시각`.
**대상이 바뀌면 재승인** — 특히 **승인한 SHA ≠ 실행할 SHA면 push·deploy를 대기**시킨다.

## ★한계 (반드시 명시)
스킬은 **조언 계층**일 뿐 강제장치가 아니다. 이 게이트는 "멈춰서 확인"을 유도할 뿐 물리적으로 막지 못한다. **실제 차단은 도구의 permission 설정 · hook · branch protection으로 병행**해야 한다.

> 왜: 일반 push와 force-push·prod 배포를 같은 무게로 승인받으면 정작 위험한 것에서 경계가 무뎌진다. 등급을 나누고 승인을 SHA·환경에 묶으면 승인이 위험에 비례한다.
