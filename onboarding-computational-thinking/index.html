내 문제를 네 조각으로

김주영 · 디자인(2022~) · 아이펠 온보딩 Computational Thinking

1. 고른 문제

바이브 코딩을 할 때, AI가 설명에 쓰는 개발 용어를 몰라서 자꾸 멈추게 된다.

나는 2022년부터 디자인 일을 해왔고 개발은 배운 적이 없다. 요즘 AI로 직접 웹을 만드는데, 답변에 모르는 용어가 나오면 그때마다 검색하느라 흐름이 끊긴다. 매번 반복되는 일이라 이걸 먼저 다뤄보기로 했다.

2. 네 동작으로 쪼개기

분해
  1. AI 답변에서 모르는 용어를 골라낸다
  2. 용어를 종류별로 나눈다 (구조 HTML / 표현 CSS / 동작·상태 JS / 도구 / 개념)
  3. 뜻을 내가 아는 말, 디자이너가 쓰는 비유로 바꿔 적는다
  4. 지금 작업에 당장 필요한지, 나중에 봐도 되는지 가른다
  5. 용어집에 적어둔다 (이미 있으면 합친다)
  6. 다음에 같은 용어가 나오면 다시 찾아 쓴다
패턴
  • 막히는 건 대부분 동작·상태(JS) 쪽이다. 색·간격 같은 시각 용어(CSS)는 비교적 금방 이해된다.
  • 같은 종류의 용어가 뭉쳐서 반복해 나온다.
  • 거의 처음 한 번만 막힌다. 적어두면 다음엔 안 막힌다.
추상화용어를 그때그때 검색하지 말고, 내 말로 번역한 용어집을 키워서 같은 용어에 두 번 막히지 않게 한다.
알고리즘AI 답변 → 모르는 용어 뽑기 → 종류 나누기 → 내 말로 번역 → 필요·보류 판단 → 용어집에 등록(중복은 합침) → 작업 계속 + 용어집 갱신

3. AI에게 전달하는 형식

[맥락]
나는 디자인 4년차(2022년 시작)이고 개발은 배운 적이 없다. AI로 직접 웹을
만드는데, 답변에 나오는 개발 용어를 몰라 작업이 자주 끊긴다. 이걸 정리하고 싶다.

[네 동작]
1. 분해   모르는 용어 뽑기 → 종류 나누기 → 내 말로 번역 → 필요·보류 판단
          → 용어집 등록 → 재사용
2. 패턴   주로 동작·상태(JS)에서 막힘 / 같은 종류가 반복 / 처음 한 번만 막힘
3. 추상화 검색을 반복하지 말고, 내 말로 번역한 용어집을 키운다
4. 알고리즘 답변 → 용어 뽑기 → 분류 → 번역 → 판단 → 등록(중복 합침) → 갱신

디자이너가 읽는 용어집 항목으로. (용어 / 한 줄 뜻 / 디자인 비유 / 언제 나오는지)
한국어로.
