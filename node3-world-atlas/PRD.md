## ① 서비스 한 줄

| 항목 | 내용 |
|---|---|
| **무엇을** | 전 세계 국가의 국기·수도·인구·면적·언어·종교를 한눈에 보는 **국가 정보 도감** |
| **누구를 위해** | 여행·유학·시사 공부 등으로 다른 나라 기본 정보를 빠르게 찾고 싶은 사람 |

한 줄 요약: **"나라 이름이나 대륙으로 찾아, 국기와 핵심 지표를 카드로 훑고, 눌러서 상세를 보는 도감."**

## ② 프레임워크 판단

**AI에게 준 조건 프롬프트:**
```
나는 전 세계 국가 정보를 보여주는 도감 사이트를 만들려고 해.
- 화면: 국가 목록, 국가 상세, (나중에) 두 나라 비교
- 나라 이름·대륙 검색으로 유입되면 좋겠어 (검색 노출 중요)
- 비전공자라 한국어 자료·예제가 많은 프레임워크를 선호해
- 지금은 백엔드 없이 목업 데이터로 만들고, 나중에 '즐겨찾기 저장'·
  '실시간 인구 API' 같은 기능을 백엔드로 붙일 계획이야

이 조건에 맞는 프레임워크 후보를 2~3개, 장단점과 "이런 경우엔 오히려 별로"까지 추천해줘.
```

**AI 추천 요지:** Next.js(검색 노출·자료 많음), Astro(콘텐츠 중심 초경량), Vue/Nuxt(점진적 학습).

**왜 이걸 골랐나 — Next.js (3문장):**
> Next.js를 골랐다. ① 나라 이름·대륙 검색 유입이 중요한데 **검색 노출(서버렌더링·정적 생성)에 유리**하고, ② React 기반이라 **한국어 자료·예제·AI 코드 생성이 가장 풍부**하며, ③ 나중에 즐겨찾기 저장·실시간 인구 API 같은 **백엔드 기능을 같은 틀 안에서 붙이기 쉽다**고 판단했기 때문이다.

**△ 고려한 대안:** 순수 콘텐츠 위주라면 **Astro**(초경량·정적)가 더 가벼운 선택. 다만 나중에 백엔드·동적 기능을 붙일 계획이라 Next.js를 우선함. (지금 만든 프로토타입은 개념 검증용이라 순수 HTML/CSS/JS로 구현 — 프레임워크는 실제 구축 단계의 계획.)

## 🎁 보너스 — 실제 페이지

과제는 PRD까지지만, 위 PRD·목업 데이터로 **목록·상세 화면을 실제 작동하게** 만들어 봄 → [index.html](index.html)
- **목록**: 국가 카드 그리드 + 이름 검색 + 대륙 필터 + 결과 수·빈 화면
- **상세**: 카드 클릭 → 전체 정보 모달
- 강의 개념 실전: **컴포넌트**(카드 재사용) · **상태**(검색·필터 → 목록·카운트·빈화면 함께 갱신) · **목업 데이터** · **목록→상세**
