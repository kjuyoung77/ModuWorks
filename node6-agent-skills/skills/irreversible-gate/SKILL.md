# 비가역 실행 게이트

"돌이킬 수 없는 것을 먼저 확인한다"는 규율. 실행 직전에 판별한다.

1. **가역/비가역 판별** — 대상이 로컬·재생성 가능(가역)인지, **비가역 바닥선**(git push · 외부 배포 · 삭제 · 개명 · 외부 발송 · 대금 확정)인지 판단한다.
2. **자율 vs 승인 구분** — 검증 통과한 로컬·재생성 가능 작업은 자율 진행 / **상업·대외·파괴적 액션은 결정권자 승인** 필요.
3. **게이트 처리** — 게이트에 걸리면 실행 직전 승인을 확보하고, 아니면 진행 후 사후 기록한다.
4. **명시** — 실행 대상·영향·되돌림 가능성을 한 줄로 적은 뒤 진행 또는 대기한다.

> 왜: 편해질수록 위험한 액션을 무심코 실행하기 쉽다. 이 게이트를 스킬로 두면 비가역 액션마다 자동으로 한 박자 멈추게 된다.
