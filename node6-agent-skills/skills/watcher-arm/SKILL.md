# 백그라운드 감시기 무장 (단일 인스턴스 계약)

파일·폴더 변화를 감지하는 감시기를 **정확히 하나만** 돌린다. 중복 감시기는 중복 알림·경쟁을 부른다.

## 단일 인스턴스 계약 (P1)
1. **lease/lock 획득 시도** — 감시 대상별 lease 파일 하나를 기준으로 한다.
2. **PID·프로세스 실존 확인** — lease에 적힌 PID가 실제로 살아있는지 확인한다(`kill -0`).
3. **live면 재사용** — 살아있는 lease면 새로 켜지 않고 기존 것을 재사용한다.
4. **stale만 회수** — 죽은(=mtime이 폴링 주기 × 2를 넘긴) lease만 회수한다. 살아있는 lease는 절대 선점하지 않는다.
5. **atomic heartbeat** — 임시 파일에 쓰고 원자적으로 교체(`mv -f`)해 부분쓰기를 막는다.
6. **정상 종료 시 lease 반납** — 종료 시 lease를 지운다(`trap ... EXIT`).

## heartbeat에 기록할 것
`writer_id/PID` · 시작 시각 · 마지막 tick · 폴링 주기 · 감시 스코프. **생존 판정은 표시 시각이 아니라 파일 mtime/epoch로.**

## 결정론적 헬퍼
설명을 매번 반복하지 말고 동봉한 `scripts/ensure_single_watcher.sh`를 쓴다.
⚠️ 공개 저장소이므로 헬퍼에 **비공개 경로·키·토큰을 넣지 않는다** — 감시 경로·스코프는 **인자로 주입**한다.

```
scripts/ensure_single_watcher.sh <감시대상경로> <lease파일경로> [폴링초=15]
```

> 왜: "감시기 켜"를 매번 손으로 짜면 lease·PID·mtime 처리를 빠뜨려 좀비·중복이 생긴다. 계약을 스킬 + 헬퍼로 두면 한 줄 요청으로 동일한 안전 형태를 재현한다.
